프로젝트 개요
프로젝트 이름: Quest_K_App 
앱 이름: Quest_K 
목적: 
 1) 한국을 여행하는 외국인들의 한국 여행을 돕는 퀘스트 어플
  - 한국 여행을 위해 필요한 사항들을 돕는다.
  - 한국 여행이 기억에 남도록 시각화 자료나 영상 등의 서비스를 제공한다.
 2) 한국의 지역 균형 발전을 위하여 한국의 모든 지역을 여행 할 수 있도록 유도
  - 한국의 7개 지역(서울, 경기, 강원, 충청, 전라, 경상, 제주) 미션을 제공
 3) 고객으로부터 데이터를 수집하여 인싸이트를 만드는 주식회사로 성장

수익화:
 1) 유저가 무료로 사용 할 수 있는 어플이며, 이용에 방해 되지 않도록 구글 애드몹(AdMob) 배너 광고 적용
 2) 배너 없이 이용하고 싶은 유저를 위한 유료 버전의 어플을 출시 예정

요구사항:
 1) 영어와 한글을 병행 표기
  - 영어를 쓰지 않는 외국인을 위해 픽토그램 등으로 파악 가능 하도록 반영
 2) 수집하는 데이터를 체계적으로 관리 및 트렌드 분석
  - 선호하는 여행: 
    🍱 Food, 🏰 History, 📸 Photo Spot, 🌳 Nature, 🛍️ Shopping. K-Culture
 3) 모든 파일들은 용도를 명확히 알 수 있도록 체계적으로 파일 관리
 4) 진행되는 모든 단계는 한글로 설명해줘
 5) 무료버전에서는 네이버지도 웹이나 앱으로 접속 할 수 있는 경로를 제공한다. (API 연동 없이 단순 링크 활용)
 
참고사항:
 1) 웹을 먼저 만들 예정이고, 만들어진 웹을 기반으로 플루토 프로그램을 통해 앱으로 만들 예정
 2) 유료 버전의 어플을 출시 예정
 3) 생성된 모든 폴더와 파일에 대한 이름과 용도에 대하여 정리된 문서를 하나 만들어줘
 4) 진행된 내용들에 대한 히스토리를 확인 할 수 있도록 시간과 진행된 사항들을 정리한 파일을 만들어서 시간순으로 관리
 5) 폴더와 파일의 구조를 파악 할 수 있도록 전체 구조를 정리한 문서를 하나 만들어줘 이름은 : structure 으로 Quest_K_App 폴더 안에 만들어줘

미션 인증 방식:
 1) 퀴즈 대신 **사진 첨부** 및 **후기(Review)** 작성으로 변경
 2) 난이도 체크(Easy/Normal/Hard) 기능을 포함하여 유저 경험 수집

1. 디자인 시스템 (Design Tokens)
 메인 컬러 (Primary): 한국적 단청의 붉은색(#D32F2F) 또는 차분한 쪽빛(Navy, #1A237E).
 포인트 컬러 (Accent): 금색(#FFD700) 또는 깨끗한 백색(#FFFFFF).
 폰트 (Typography): * 영문: 고전적이고 신뢰감을 주는 'Serif' 계열과 가독성이 좋은 'Sans-serif' 혼용.

한글: 나눔명조 또는 본고딕 (무료 폰트 권장).
컴포넌트 스타일: 둥근 모서리(Border Radius: 12px~16px)를 사용하여 현대적이고 부드러운 인상을 줍니다.

2. 데이터 구조 (Database Schema)

 1) Users Collection (사용자):
  - uid (고유 ID)
  - email (이메일)
  - nickname (닉네임)
  - visit_count (방문 횟수: 1, 2, 3, 4, 5+)
  - interests (관심사 리스트)
  - unlocked_badges (획득한 배지 리스트)
 2) Quests Collection (미션):
  - quest_id (미션 고유번호)
  - region (권역명: 서울, 제주 등)
  - difficulty_avg (유저들이 선택한 평균 난이도 저장 필드)
 3) Reviews Collection (후기):
  - review_id (고유 ID)
  - user_id (작성자)
  - quest_id (대상 미션)
  - photo_url (인증 사진)
  - comment (사용자 후기)
  - difficulty_rating (사용자가 느낀 난이도)
  - created_at (작성 시간)

3. 핵심 기능 로직 (Logic Flow)

 미션 완료 프로세스: 
 1. 사진 업로드
 2. 후기 작성 및 난이도 체크
 3. 완료 후 포인트 및 경험치 지급

 마이 퀘스트북 자동 생성: 완료된 미션 데이터만 필터링하여 타임라인 형태로 나열.

4. 반응형 레이아웃 정책 (Responsive Design)

 모바일 우선 (Mobile First): 여행 중 현장에서 사용하는 서비스이므로 세로형 UI를 기본으로 설계.
 태블릿/웹 확장: PC 웹으로 접속했을 때는 마이 퀘스트북을 넓게(Grid 방식) 볼 수 있도록 레이아웃 그리드 설정.

5. 외부 서비스 연동 계획 (Integrations)

 인증: Firebase Auth (Google, Apple 로그인).
 이미지 저장: Firebase Storage (사용자가 찍은 인증 사진 저장용).
 데이터베이스: Firebase Firestore.
 광고: Google AdMob (모바일 앱 전환 시).
 지도: Naver Map (단순 URL 스킴/웹 링크 연동).
