# Quest K 프로젝트 진행 현황 보고서

본 문서는 Quest K App 프로젝트의 최신 작업 내역 및 개편된 요구사항을 바탕으로 작성된 보고서입니다. 

---

## 1. 개편된 주요 반영 사항 (Pivot Updates)

프로젝트의 실질적인 유저 경험과 수익성을 고려하여 다음과 같은 핵심 로직 변경을 반영하였습니다.

- **미션 인증 방식의 혁신**: 단순 퀴즈(Quiz) 풀기에서 벗어나, **'인증 사진 업로드 + 텍스트 후기 작성 + 체감 난이도 체크'** 방식으로 변경하여 데이터 신뢰도를 높이고 유저 참여를 유도했습니다.
- **데이터 구조 최적화**: 체류 기간(Stay Duration) 수집 기능을 삭제하고, 여행 스타일 및 미션 피드백 데이터 수집에 집중하도록 데이터 베이스 설계를 간소화했습니다.
- **실전형 부가 기능 조정**: 지도 API 연동이나 사전 준비 미션 등 실 사용성이 떨어지는 항목을 제거하고, 네이버 지도 웹/앱 직접 연결 방식을 채택하여 최적의 성능을 확보했습니다.
- **수익화 및 마케팅**: 광고 시스템을 구글 **AdMob** 배너 형태로 확정하여 모바일 앱 전환 시의 수익 경로를 명확히 했습니다.

## 2. 현재 작업 완료 사항 (Status: Green)

### ✅ 핵심 기능 및 UI
1.  **미션 워크플로우 재설계**: `QuestDetail` 페이지에서 사진-후기-난이도로 이어지는 3단계 프로세스 완료.
2.  **데이터 클렌징**: 불필요한 퀴즈 데이터 및 사전 준비/ESG 관련 미션 데이터 제거.
3.  **다국어 시스템 (KO/EN)**: 모든 새로운 UI 컴포넌트에 한/영 병행 표기 적용.
4.  **사용자 성장 시스템**: 경험치(EXP) 획득 및 레벨링 서비스 로직 안정화.

### ✅ 문서 인프라 (Documentation)
1.  **`structure.md`**: 전체 프로젝트 폴더 및 파일 명세를 정의한 문서 구축.
2.  **`HISTORY.md`**: 초기 개발부터 현재까지의 마일스톤을 시간순으로 정리.
3.  **`INSTRUCTIONS.md`**: 최신 요구사항을 반영한 프로젝트 설계서 업데이트.

## 3. 백엔드 연동 및 향후 로드맵

### 🚀 차후 진행 예정 사항
1.  **Firebase 실제 연동 (핵심)**:
    - **Authentication**: Google/Apple 소셜 로그인.
    - **Firestore**: 유저 데이터, 미션 완료 이력, 후기 데이터 저장.
    - **Storage**: 유저가 업로드한 인증 사진 파일 저장.
2.  **AdMob 배너 공간 확보**: 모바일 앱 래퍼(Pluto 등) 적용 시 노출될 광고 슬롯 디자인 반영.
3.  **유료 버전 멤버십 로직**: 광고 제거 및 전용 뱃지 부여 시스템 설계.
